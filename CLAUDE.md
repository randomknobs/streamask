# streamask

Генеративная AR-маска для стрима. Работает в браузере, живёт на GitHub Pages.
MediaPipe Face Landmarker (трекинг лица) + three.js (рендер поверх видео с вебки).

Полная спецификация — `SPEC.md`. Работа идёт по фазам, описанным там.

## Как запустить

```bash
python3 -m http.server 8000
```

Открыть `http://localhost:8000`. Камера требует secure context — по `file://`
работать не будет никогда, только `localhost` или https.

Прод: `https://randomknobs.github.io/streamask/`

**Сервер запускает пользователь сам, в своём терминале.** Не запускать его из
агента, не отправлять в фон, не использовать `&`, `&&`, `;` и пайпы в одной
команде — такие строки не проходят статический анализ разрешений и на каждой
требуют ручного подтверждения, что ломает работу.

Правило простое: одна команда — один вызов. Нужно проверить, что файл отдаётся —
один `curl`, отдельным вызовом. Нужно закоммитить — `git add`, потом `git commit`,
потом `git push`, тремя вызовами, а не цепочкой.

## Стек и ограничения

- Никаких сборщиков, бандлеров, npm, node_modules. Только нативные ES-модули
  через importmap с CDN. GitHub Pages отдаёт файлы как есть, любой build-шаг
  ломает деплой.
- three.js и `@mediapipe/tasks-vision` подключаются с jsdelivr, версии
  зафиксированы в importmap в `index.html`. Не обновлять их походя.
- Никаких ассетов: текстур, моделей, шрифтов. Всё процедурно, в коде.
- Не переписывать на React или любой фреймворк.

## Главное правило: детерминизм

Вся генерация идёт от строкового сида. Один сид всегда даёт одну и ту же маску,
независимо от того, сколько масок сгенерировано до этого в сессии.

- `Math.random()` допустим **только** при выборе нового случайного сида
- Везде остальное — `rng.js` (mulberry32), пересеваемый в начале `generate()`
- Никакого общего мутирующего состояния между слоями без явного пересева
- Сид пишется в `location.hash` и восстанавливается оттуда при загрузке

Если рекол сохранённой маски даёт другой результат — это баг, а не мелочь.

## Архитектура

```
index.html          разметка, importmap
src/main.js         bootstrap, главный цикл, склейка слоёв
src/tracking.js     MediaPipe, face basis, сглаживание, blendshapes
src/rng.js          mulberry32, seedFrom, хелперы R/RI/pick
src/palette.js      цветовые схемы и семейства
src/uv.js           статические UV канонического меша (468 точек)
src/layers/*.js     слои: skin, shards, mouth, eyes, brows
src/texture/*.js    паттерны, зоны лица, сборка шейдера
src/storage.js      сохранение масок в localStorage
src/record.js       запись видео через MediaRecorder
src/ui.js           панель, хоткеи
```

Контракт слоя — единый:

```js
export function create(ctx) // ctx: { scene, rng, palette, params }
                            // → { object3D, update(state, dt), dispose() }
```

`state` каждый кадр: `{ landmarks, blendshapes, basis, t, audio? }`.

Новый визуальный элемент = новый слой по этому контракту, а не правка чужого файла.

## Система координат

Ортографическая камера в normalized-space видео, чтобы маска совпадала с
картинкой при любом FOV. Лендмарки маппятся так (зеркально по X, селфи-режим):

```js
x = -(lm.x - 0.5) * 2 * aspect
y = -(lm.y - 0.5) * 2
z = -lm.z * 2 * aspect
```

`anchor` — группа с матрицей, собранной из базиса лица (точки 33, 263, 10, 152).
Единица длины в её локальных координатах = межзрачковое расстояние. Всё, что
крепится к лицу, живёт внутри анкера и не зависит от расстояния до камеры.

UV меша кожи — **статические**, из канонической модели. Никогда не брать UV из
живых лендмарков: они плывут при повороте головы и любой регулярный паттерн
поедет.

## Правила коммитов

- Коммитить в `main` напрямую и пушить после каждой фазы
- Одна фаза = один коммит, сообщение вида `phase 3a: static canonical UVs`
- Не начинать следующую фазу, пока предыдущая не проверена в браузере
- Не смешивать рефакторинг и новую фичу в одном коммите

Команды git разрешены в `.claude/settings.json` и проходят без подтверждения —
но только по одной за вызов, без цепочек.

## Приёмка любой фазы

Прежде чем считать фазу готовой, проверить:

1. 30+ fps на встроенной вебке
2. Маска держится при повороте головы ±45° и не дёргается
3. Один сид воспроизводит идентичную маску после перезагрузки страницы
4. Хоткеи живы: R — новая маска, H — скрыть панель
5. Консоль чистая, без ошибок и варнингов three.js
6. Сгенерировать 10 масок подряд — все визуально разные

## Чего не делать

- Не воспроизводить конкретные существующие произведения искусства и узнаваемых
  персонажей. Референсы — источник принципов (плоские заливки, зонирование лица,
  жёсткий контур поверх цвета), а не материал для копирования.
- Не добавлять аналитику, телеметрию, внешние запросы. Страница работает
  полностью локально, кроме загрузки библиотек и модели MediaPipe с CDN.
- Не удалять и не «упрощать» уже работающие слои ради своей реализации —
  расширять.
