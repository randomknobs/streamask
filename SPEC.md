# streamask — спецификация v2

Генеративная AR-маска для стрима. Браузер, MediaPipe Face Landmarker + three.js.
Текущее состояние: один файл `index.html`, два слоя (тесселированная «кожа» с
градиентным шейдером + процедурные «осколки» вокруг лица), генерация от сида,
сид в `location.hash`.

Цель v2: плотная, фактурная, эмоциональная маска вместо гладкого градиента.

---

## Правила работы

- Коммитить в `main` напрямую, пушить после каждой фазы.
- Одна фаза = один коммит. Сообщение: `phase N: краткое описание`.
- Не переходить к следующей фазе, пока предыдущая не работает в браузере.
- Не ломать существующее: сид всегда воспроизводит одну и ту же маску, hash в URL
  продолжает работать, панель UI и хоткеи (R, H) остаются.
- Никаких сборщиков и npm. Только ES-модули с CDN, чтобы GitHub Pages отдавал как есть.
- После каждой фазы прогнать глазами: 30+ fps на встроенной вебке, маска не
  отваливается при повороте головы ±45°.

```js
export function create(ctx)      // ctx: { scene, rng, palette, params }

```

Приёмка: визуально ничего не изменилось, тот же сид даёт ту же маску.

## Фаза 1 — плотность

Сейчас элементов мало и они разлетаются. Надо много и кучно.

- Количество осколков: `RI(14,4)` → `RI(60,22)`. Инстансинг через
  `InstancedMesh` по группам одинаковой геометрии, иначе просядет fps.
- Разброс: `px` до `R(1.1,.05)`, `py` до `±0.9`. Держатся у лица, не по всему кадру.
- Ввести три пояса плотности с разными правилами размера:
  - `core` (r < 0.5) — мелкие, много, плотно, почти без зазоров
  - `mid` (0.5–1.0) — средние
  - `halo` (1.0–1.4) — крупные, редкие, полупрозрачные
- Кластеризация: 3–6 «гнёзд» на лице (случайные точки), 70% элементов
  сэмплится вокруг гнёзд с гауссовым разбросом, 30% равномерно. Это даёт
  скопления вместо ровного шума.
- Симметрия остаётся, но добавить `asymmetryBias` 0..0.3 — доля элементов,
  которые намеренно не зеркалятся. Живее выглядит.

Приёмка: маска читается как единый объект, а не как редкие висящие фигуры.

## Фаза 2 — рот, глаза, брови, реакция на мимику

### 2a. Blendshapes

В `createFromOptions` включить `outputFaceBlendshapes: true`. Прокинуть в `state`
как объект `{ jawOpen, mouthPucker, mouthSmileLeft, ..., eyeBlinkLeft, browDownLeft, ... }`
(значения 0..1, по имени категории).

Сгладить экспоненциально: `v += (target - v) * 0.35`, иначе дрожит.

### 2b. Движение от рта

Завести `mouthEnergy = jawOpen*0.7 + mouthFunnel*0.2 + mouthPucker*0.1`.
Повесить на:
- масштаб осколков: `scale = base * (1 + mouthEnergy * k)`, где `k` у каждого
  элемента свой, из сида, в диапазоне 0..0.8 — часть не реагирует вовсе,
  часть скачет сильно
- радиальное смещение: при открытом рте элементы `core`-пояса разъезжаются
  от центра на `mouthEnergy * 0.15`
- скорость вращения: `spin *= (1 + mouthEnergy * 2)`
- у части элементов (флаг из сида) — инверсия: сжимаются при открытии рта

Смысл: маска должна «говорить» вместе с человеком.

### 2c. Шум в дырке рта

Внутренний контур губ, 20 точек, по порядку кольца:

```
78, 95, 88, 178, 87, 14, 317, 402, 318, 324,
308, 415, 310, 311, 312, 13, 82, 81, 80, 191
```

(порядок проверить эмпирически — если полигон самопересекается, развернуть).

Построить `BufferGeometry` веером от центроида кольца: `N` треугольников
`[centroid, ring[i], ring[i+1]]`. Позиции обновлять каждый кадр из лендмарков,
z смещать на `-0.004` (чуть в глубину, за плоскость кожи).

Материал — `ShaderMaterial`, аналоговый ТВ-шум:

```glsl
// uniforms: t, intensity, tint, rollSpeed
float n = hash(vec2(floor(vUv.x * 220.), floor(vUv.y * 160.) + floor(t * 24.)));
float scan = 0.85 + 0.15 * sin(vUv.y * 400. + t * 6.);        // строчная сетка
float roll = smoothstep(0.0, 0.08, fract(vUv.y - t * rollSpeed)); // бегущая полоса
vec3 c = vec3(n) * scan * mix(0.6, 1.0, roll);
c = mix(c, c * tint, 0.5);                                     // лёгкий цвет от палитры
```

Шум должен обновляться дискретно (24 кадра в секунду через `floor(t*24)`), а не
плавно — иначе выглядит как рябь, а не как телевизор.

Интенсивность привязать к `jawOpen`: рот закрыт — шум глушится до нуля.

### 2d. Глаза — объекты вместо дырок

Кольца глаз:

```
right: 33, 7, 163, 144, 145, 153, 154, 155, 133, 173, 157, 158, 159, 160, 161, 246
left : 263, 249, 390, 373, 374, 380, 381, 382, 362, 398, 384, 385, 386, 387, 388, 466
```

Центр глаза брать по радужке: точки 468–472 (правый) и 473–477 (левый) —
`FaceLandmarker` отдаёт 478 точек, последние 10 это ирисы.

На каждый глаз собирается процедурная «сборка» из 2–5 элементов, привязанная к
центру глаза, масштабируется по ширине кольца:

- концентрические кольца (`RingGeometry`) со случайным количеством и толщиной
- спираль (`TubeGeometry` по `CatmullRomCurve3`)
- гранёный многоугольник, залитый плоским цветом
- пучок «ресниц» — тонкие конусы по радиусу, случайное количество
- вложенный «зрачок» — сфера или диск контрастного цвета

Ключевое: **левый и правый глаз генерируются с разными сидами**
(`seed` и `seed ^ 0x9E3779B9`). Асимметрия глаз — половина выразительности.

Реакция: `eyeBlinkLeft/Right` схлопывает сборку по Y до 0.15 и обратно.
На `jawOpen` глаза слегка раскрываются (масштаб +10%).

Дырки под настоящие глаза в слое кожи можно оставить — сборка их перекроет.

### 2e. Брови — эмо-настрой

Брови **не привязаны** к реальным бровям, это фиксированный характер маски.
Ставятся над глазными кольцами в face-local координатах.

Пресеты (выбирается один на маску из сида, равновероятно):

| пресет | форма | наклон внутр. конца | толщина |
|---|---|---|---|
| `angry` | прямая, клин | вниз к переносице, −25° | толстая |
| `sad` | прямая | вверх к переносице, +22° | тонкая |
| `surprised` | высокая дуга | горизонт | средняя |
| `skeptical` | одна вверх, вторая вниз | асимметрия 30° | средняя |
| `manic` | зигзаг, 3–4 излома | +35° | толстая |
| `dead` | горизонтальная палка | 0° | очень тонкая |
| `unibrow` | одна сплошная через переносицу | 0° | очень толстая |

Реализация: `Shape` → `ExtrudeGeometry` с небольшой глубиной, либо
`TubeGeometry` по `CatmullRomCurve3` из 4–6 контрольных точек. Второе гибче
для зигзага.

Цвет — контрастный к коже (взять из палитры точку с максимальной разницей по
светлоте). Имя пресета показывать в панели рядом с сидом, это часть удовольствия.

Небольшая жизнь: `browDownLeft/Right` из blendshapes добавляет ±8° к наклону.
Не подмена пресета, а лёгкая модуляция.

## Фаза 3 — фактура кожи

Самая важная фаза. Сейчас гладкий градиент, нужна графика.

### 3a. Статические UV

Критично. Сейчас `uv = [landmark.x, 1 - landmark.y]` — координаты плывут при
повороте головы, любая клетка поедет. Нужны фиксированные UV канонического меша.

Взять UV из `canonical_face_model.obj` (репозиторий mediapipe,
`mediapipe/modules/face_geometry/data/`), выгрузить в `src/uv.js` как
`Float32Array` длиной 468*2, проставить один раз при создании геометрии,
больше не трогать.

Для точек 468–477 (ирисы) UV не нужны — они в меш кожи не входят.

### 3b. Слоистый генератор текстуры

Вместо одного шейдера — стек из 2–4 слоёв, каждый со своим паттерном, режимом
смешивания и маской зоны. Собирается из сида.

Паттерны (`src/texture/patterns.glsl.js`, каждый — функция `float pattern(vec2 uv)`):

- `checker` — клетка, масштаб 4..40, опционально неравномерная (разный шаг по осям)
- `checkerNested` — клетка в клетке, три уровня разного размера
- `stripes` — полосы, угол произвольный, толщина неравномерная
- `voronoiFlat` — ячейки Вороного, плоская заливка случайным цветом из палитры
  → это и есть «кубизм», основной инструмент для фасеточного вида
- `voronoiEdge` — только границы ячеек, тонкие линии
- `dots` — сетка точек разного размера, гексагональная или квадратная
- `concentric` — концентрические кольца от произвольного центра
- `beads` — плотные ряды кружков вдоль кривых, имитация бисера
- `hatch` — штриховка в два перекрёстных направления
- `blobs` — крупные органические пятна (fbm + threshold), под плоскую заливку
- `zigzag` — угловатый орнамент, ряды треугольников

Режимы смешивания: `over`, `multiply`, `screen`, `mask` (слой виден только там,
где нижний прошёл порог), `outline` (только края нижнего слоя).

### 3c. Зоны лица

Тут появляется карнавальность: разные зоны — разная обработка.

Разметить UV-пространство на зоны и хранить как маску (проще всего — заранее
посчитанная текстура 512×512 с индексом зоны в каналах, либо аналитические
эллипсы в UV-координатах, что дешевле и достаточно):

- `forehead` — лоб
- `eyeBandLeft` / `eyeBandRight` — области вокруг глаз
- `noseBridge` — переносица
- `cheekLeft` / `cheekRight` — щёки
- `mouthArea` — вокруг рта
- `chin` — подбородок
- `jawline` — контур челюсти

Схемы раскладки (выбирается одна из сида):

- `mirror` — левая и правая половина одинаковые
- `split` — левая и правая половина принципиально разные (разный паттерн и палитра)
- `bands` — горизонтальные полосы по высоте лица, каждая со своим паттерном
- `mask-over-eyes` — акцентная полоса через глаза, остальное однотонное
- `radial` — паттерн расходится от кончика носа
- `patchwork` — каждая зона получает свой паттерн независимо

### 3d. Контурные линии

В рефах везде есть жёсткая графика поверх заливок. Добавить слой линий:
- обводка границ зон толщиной 2–5 px (в UV — через `fwidth` для стабильной толщины)
- случайные орнаментальные дуги и штрихи по зонам
- цвет — самый тёмный или самый светлый из палитры

Без этого слоя всё будет выглядеть как размытые пятна.

### 3e. Палитры

Расширить `palette.js`. Помимо алгоритмических схем добавить именованные
семейства с характерным настроением — берутся как ограничения на HSL, а не как
фиксированные списки цветов, чтобы внутри семейства всё равно был разброс:

- `opera` — насыщенный красный + кобальт + белила + золото, высокий контраст
- `beadwork` — бирюза, алый, кремовый, чёрный; средняя светлота, высокая насыщенность
- `surreal` — большое поле одного холодного цвета + 2–3 маленьких ярких акцента
  (красный, чёрный, розовый), очень контрастно по площади
- `earth` — охра, терракота, уголь, кость
- `neon` — кислотные на почти чёрном
- `bw` — только чёрный, белый и один цвет

Важно: это направления настроения, а не копирование конкретных работ. Каждая
маска должна оставаться сгенерированной, а не воспроизведённой.

Пропорции площадей задавать явно: доминирующий цвет 60–70%, второй 20–30%,
акценты 5–10%. Равные доли выглядят вяло.

Приёмка фазы 3: 10 подряд сгенерированных масок — все разные по типу фактуры,
ни одна не выглядит как «градиент с шумом».

## Фаза 5 — сохранение и рекол

`src/storage.js`, `localStorage`, ключ `streamask:v1`.

Хранить **только параметры генерации**, не геометрию:

```json
{
  "version": 1,
  "saved": [
    { "seed": "a7f3k2", "name": "злой карнавал", "ts": 1750000000000,
      "thumb": "data:image/jpeg;base64,...", "params": { "density": 1.2 } }
  ]
}
```

- Кнопка «сохранить» — снимает превью 160×160 с канваса (jpeg, quality 0.6),
  кладёт в список
- Галерея превью в панели, клик — мгновенный рекол маски
- Переименование по двойному клику, удаление по кнопке
- Экспорт/импорт всей коллекции в json-файл (на случай очистки браузера)
- Следить за квотой: `localStorage` это ~5 МБ, при 50+ превью упрётся.
  Ограничить список 40 записями, при переполнении — предупреждение, а не молчаливая
  потеря. Превью хранить не больше 160 px.

Проверить, что рекол воспроизводит маску **точно**: один сид → одна маска,
независимо от того, сколько масок сгенерировано до этого. Значит `rnd`
обязан пересоздаваться в начале `generate()`, никакого шаренного состояния между слоями
без явного пересева.

## Фаза 6 — запись видео

`src/record.js`.

Проблема: сейчас `<video>` лежит под прозрачным канвасом, это два разных
элемента. Записать так нельзя.

Решение — композитный канвас:

1. Создать `canvas#out` нужного размера (по умолчанию 1280×720)
2. Каждый кадр: `ctx.drawImage(video, ...)` с зеркалированием, затем
   `ctx.drawImage(glCanvas, ...)`
3. `out.captureStream(30)` → `MediaRecorder`
4. Кодек: `video/webm;codecs=vp9`, фолбэк на `vp8`, затем на дефолт.
   Проверять через `MediaRecorder.isTypeSupported`
5. Стоп → `Blob` → `URL.createObjectURL` → `<a download>` с именем
   `streamask-<seed>-<timestamp>.webm`

Дополнительно:
- Кнопка записи + хоткей, индикатор (красная точка) и таймер
- Опционально дорожка микрофона: `new MediaStream([...videoTracks, ...audioTracks])`.
  Отдельный тумблер, по умолчанию выключен
- Битрейт: `videoBitsPerSecond: 8_000_000`
- Режим «прозрачный фон» — не рисовать видео в композит, только маску. VP9 в webm
  умеет альфу через `codecs=vp09.00.10.08` не везде; если не поддерживается,
  писать на зелёном фоне и сказать об этом в UI
- Не держать чанки в одном массиве бесконечно — при длинной записи предупредить
  на 10 минутах

Проверить, что композит не роняет fps ниже 30. Если роняет — писать в
`requestVideoFrameCallback` вместо основного лупа.

### 6b — кнопка «поделиться»

Работает поверх композитного канваса из 6а, поэтому делается вместе
с записью, а не отдельно.

**Снимок**

```js
const blob = await new Promise(r => outCanvas.toBlob(r, 'image/png'));
const file = new File([blob], `streamask-${seed}.png`, { type:'image/png' });
```

Композит для снимка собирается тем же путём, что и для записи: видео
с зеркалированием, сверху WebGL-канвас. Панель UI в кадр не попадает —
она отдельный DOM-элемент, но проверь, что слайдеры и подписи не
оказались нарисованы внутри канваса.

**Шаринг**

```js
if (navigator.canShare?.({ files:[file] })) {
  await navigator.share({ files:[file], title:'streamask', text:`seed: ${seed}` });
}
```

Системная шторка сама предложит установленные приложения. Никаких
API-ключей, OAuth и бэкенда — всё клиентское.

**Фолбэк**

`navigator.canShare` с файлами есть не везде: работает на мобильных
и в Safari на macOS, в десктопном Chrome файлы не поддерживаются.
Определять только через `canShare({files})`, не по user-agent.
Если не поддерживается — скачивание файла через `<a download>`,
и кнопка меняет подпись на «скачать».

**Видео тоже**

Тот же механизм для записанного webm: после остановки записи кнопка
«поделиться» отдаёт видеофайл вместо png. Учти, что часть приложений
webm не принимает — если это окажется проблемой, вариант с mp4 через
`isTypeSupported('video/mp4')`, где поддерживается.

**Текст поста**

В `text` кладём сид и прямую ссылку на маску:
`https://randomknobs.github.io/streamask/#<seed>` — тогда по ссылке
из поста открывается ровно эта маска. Это и есть главная причина
писать сид в hash.

**UI**

Кнопка рядом с кнопкой записи. Одна кнопка на оба режима: если есть
свежая запись — делится видео, иначе снимает и делится кадром.
Хоткей не нужен.

**Ограничения**

- `navigator.share` вызывается только из обработчика реального клика,
  иначе браузер отклонит вызов. Никаких `await` до вызова —
  готовь blob заранее или показывай индикатор.
- Отмена шаринга пользователем прилетает как исключение `AbortError`,
  его надо глушить, а не показывать как ошибку.
  

## Фаза 7 — реакция на звук (опционально, после остального)

`AnalyserNode` от микрофона или от выбранного аудиовхода.

- FFT 2048, три полосы: низ (20–200 Гц), середина (200–2000), верх (2000+)
- Сглаживание с разным временем атаки и релиза (атака быстрая, релиз медленный)
- Детект удара: производная энергии низов выше порога → импульс
- Привязка: низ → масштаб осколков и толчок наружу; середина → скорость
  анимации паттернов; верх → интенсивность контурных линий и ТВ-шума
- На удар — кратковременный сдвиг hue всей палитры или переключение
  `checker` в инверсию на 100 мс
- Обязательно тумблер и слайдер чувствительности

## Что не делать

- Не тянуть тяжёлые библиотеки. Всё процедурно, ассетов нет вообще
- Не превращать в React-приложение
- Не воспроизводить конкретные существующие произведения или узнаваемые
  персонажи — только абстрактные генеративные формы
- Не привязывать генерацию к `Math.random()` нигде, кроме выбора нового сида.
  Всё остальное — только через сид, иначе рекол сломается
- `localStorage` не работает внутри превью артефактов в чате, это нормально —
  тестировать на GitHub Pages или на локальном сервере


  ## Фаза 8 — головные уборы (слой crown)

`src/layers/crown.js`, контракт слоя обычный. Отдельный тумблер
в панели, наравне с кожей и осколками.

### Крепление
Купол строится в face-local координатах анкера, над точкой 10 (лоб).
Радиус 1.1–1.8 IPD, чтобы перекрывать волосы сверху и по бокам.
Наклон вперёд/назад 0..20° из сида.

### Силуэтные семейства
Одно на маску, равновероятно. Это абстрактные архетипы формы,
а не реплики конкретных существующих уборов:

- `dome` — сплошной купол или полусфера, гладкий или гранёный
- `crest` — гребень вдоль макушки спереди назад, высота 0.6–1.4,
  плоский или зубчатый
- `halo` — кольцо или несколько концентрических колец над головой,
  наклон произвольный
- `rays` — венец из расходящихся лучей: конусы или клинья по кругу,
  8–24 штуки, длина неравномерная
- `tiers` — 2–4 яруса убывающего радиуса друг на друге, конус
  или ступенчатая башня
- `horns` — два крупных изогнутых элемента по бокам, вверх или в стороны
- `wrap` — обмотка: несколько лент, идущих вокруг головы под разными
  углами, перекрывающими друг друга
- `plume` — пучок вытянутых элементов, торчащих вверх и назад,
  разной длины, слегка веером

### Бахрома
Отдельный подслой, включается с вероятностью ~0.6 независимо
от семейства. Ряды тонких вытянутых элементов, свисающих вниз
вдоль висков от края купола до линии челюсти. Именно она закрывает
волосы по бокам. Количество 10–40 на сторону, длина неравномерная,
лёгкий разброс по углу.

### Декор
Поверх основы 0–12 мелких элементов из того же словаря примитивов,
что у осколков: диски, шары, кольца, гранёные многогранники.
Расставляются по силуэту, симметрично.

### Симметрия
Обязательна для основы. Асимметрия допускается только в декоре
и в длине бахромы, не более 0.2 доли элементов.

### Палитра
Общая с маской, плюс один акцентный цвет, которого нет на лице —
берётся сдвигом hue на 0.5 от доминирующего. Материалы по тем же
правилам: 60% solid, 20% glass, 20% wire.

### Реакция
На blendshapes не реагирует. Крепится жёстко к анкеру, вся жизнь —
от движения головы.

### Важно
Формы должны оставаться сгенерированными абстракциями. Не
воспроизводить узнаваемые реальные регалии, религиозные предметы
и церемониальные уборы конкретных культур — работаем на уровне
геометрических архетипов, как и весь остальной проект.
